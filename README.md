# 🎬 Кинотеатр на двоих — инструкция

Сайт, где вы с другом из другого города смотрите фильмы вместе с **общим плеером**:
один ставит на паузу — у второго тоже пауза, один перематывает — у второго тоже.
Сбоку — **текстовый чат**.

## Как это устроено (коротко)

- **Сам фильм** лежит в облачном хранилище **Cloudflare R2** (бесплатно 10 ГБ,
  бесплатная отдача). Браузеры тянут видео прямо оттуда — твой интернет не грузится.
- **Сайт + синхронизация + чат** работают на бесплатном сервере **Render**.
  По сети между вами летят только маленькие команды (play/пауза/перемотка/сообщения).

```
Ты ─┐                                   ┌─ видео из R2
    ├── Render (синхронизация + чат) ────┤
Друг ┘                                   └─ видео из R2
```

Дальше — три части: **(1)** проверить локально, **(2)** залить фильм в R2,
**(3)** выложить сайт на Render. Потом — как смотреть.

---

## Часть 0. Проверить на своём компьютере (5 минут, без интернета для друга)

Это чтобы убедиться, что всё работает, ещё до возни с хостингами.

1. Установи **Node.js** (если ещё нет): https://nodejs.org → кнопка **LTS** → установи.
2. Открой папку `cinema` в терминале (в VS Code: правой кнопкой по папке → «Открыть
   в интегрированном терминале»).
3. Введи команду и нажми Enter:
   ```
   npm install
   ```
   (скачает нужные библиотеки, один раз)
4. Запусти сервер:
   ```
   npm start
   ```
   Увидишь строчку `🎬 Кинотеатр запущен на порту 3000`.
5. Открой в браузере **две вкладки** с адресом `http://localhost:3000` (это как будто
   вас двое).
6. В **первой** вкладке: впиши имя → **Создать комнату**. Сверху появится **код**
   (например `K7Q2`).
7. Во **второй** вкладке: впиши другое имя → в поле «КОД» введи тот код → **Войти**.
   В обеих вкладках статус станет зелёным «друг в сети».
8. Тыкни **Avatar** в одной вкладке → он откроется и во второй.
   Жми play/пауза/перематывай — вторая вкладка повторяет. Напиши в чат — придёт.
   (Avatar уже подключён к каталогу и играет прямо с твоего компьютера.)

✅ Работает? Отлично. Локально всё готово.
Дальше — чтобы друг из другого города мог зайти **по ссылке**, нужно сделать ещё
два бесплатных шага: залить фильм в облако (Часть 1) и выложить сайт (Часть 2).
> Чтобы остановить сервер — в терминале нажми `Ctrl + C`.

> ⚠️ Важно: сейчас Avatar играет по адресу `/local/Avatar.mp4` — это работает
> **только на твоём компьютере**. Друг по интернету этот файл не увидит. Поэтому
> для просмотра вдвоём фильм надо залить в облако (R2) — об этом Часть 1.

## Часть 1. Залить фильм в Cloudflare R2 (бесплатно)

Твой файл 1.6 ГБ туда влезет легко (бесплатно дают 10 ГБ).

1. Зайди на **https://dash.cloudflare.com** → зарегистрируйся (email + пароль).
2. Слева в меню найди **R2** (R2 Object Storage). Нажми → если попросят, нажми
   **Add billing / подтвердить** — для бесплатного тарифа карта обычно нужна только
   для верификации, в пределах 10 ГБ денег не снимают.
3. Нажми **Create bucket** (создать «корзину»).
   - Имя, например: `kino`
   - Нажми **Create bucket**.
4. Зайди в свою корзину → вкладка **Settings** (Настройки).
5. Найди раздел **Public access** → **R2.dev subdomain** → нажми **Allow Access**
   (включить публичный доступ). Подтверди.
   - Появится публичный адрес вида `https://pub-xxxxxxxx.r2.dev` — он понадобится.
6. Вернись на вкладку **Objects** → нажми **Upload** → выбери `Avatar.mp4`.
   Дождись окончания загрузки (1.6 ГБ — несколько минут).
   > Файл должен быть **.mp4** (формат H.264) — у тебя как раз такой ✅.
   > (Если когда-нибудь будет `.mkv`/`.avi` — сконвертируй в mp4 в бесплатном
   > HandBrake, иначе видео может не открыться в браузере.)
7. Кликни на загруженный файл → скопируй его **публичную ссылку**. Это будет
   `https://pub-xxxxxxxx.r2.dev/Avatar.mp4`.

8. Открой файл [public/movies.json](public/movies.json). У фильма Avatar замени
   `"/local/Avatar.mp4"` на скопированную ссылку из R2:
   ```json
   {
     "id": "avatar",
     "title": "Avatar",
     "type": "Фильм",
     "poster": "posters/avatar.jpg",
     "videoUrl": "https://pub-xxxxxxxx.r2.dev/Avatar.mp4"
   }
   ```
   (обложку `avatar.jpg` можно положить в папку `public/posters/` позже — без неё
   просто будет значок 🎬)

> 💡 Каждый новый фильм добавляется так же: залил в R2 → скопировал ссылку →
> дописал ещё один блок `{ ... }` в `movies.json` (не забудь запятую между блоками).

## Часть 2. Выложить сайт в интернет на Render (бесплатно)

Чтобы друг из другого города мог открыть сайт по ссылке.

### 2.1 Загрузить код на GitHub
1. Зарегистрируйся на **https://github.com**.
2. Нажми **New repository** → имя, например `kino` → **Create repository**.
3. Загрузить файлы можно прямо через сайт: на странице репозитория **Add file →
   Upload files** → перетащи **содержимое папки `cinema`** (файлы `server.js`,
   `package.json`, папку `public`) → **Commit changes**.
   > Видео в GitHub заливать НЕ нужно — оно уже в R2.

### 2.2 Создать сервис на Render
1. Зайди на **https://render.com** → войди через GitHub.
2. Нажми **New +** → **Web Service**.
3. Выбери свой репозиторий `kino` → **Connect**.
4. Заполни поля:
   - **Name**: любое, например `kino`
   - **Region**: ближайший (Frankfurt подойдёт)
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: **Free**
5. Нажми **Create Web Service**. Подожди 1–2 минуты — Render всё соберёт.
6. Сверху появится адрес вида **`https://kino-xxxx.onrender.com`** — это и есть
   ссылка на ваш кинотеатр. Открой её — должен открыться сайт.

> ⚠️ На бесплатном Render сервер «засыпает» после простоя. Первый заход после паузы
> грузится ~30–50 секунд — это нормально, просто подожди. Само видео это не тормозит
> (оно из R2).

## Часть 3. Как смотреть вдвоём 🍿

1. Открой ссылку Render (`https://kino-xxxx.onrender.com`).
2. Впиши имя → **Создать комнату** → сверху появится **код** (нажми на него, чтобы
   скопировать).
3. Отправь другу **ссылку сайта + код** (в телеграм/вотсап).
4. Друг открывает ссылку → вписывает имя → вводит код → **Войти**.
5. Когда оба в сети — статус зелёный «друг в сети».
6. Любой тыкает фильм в каталоге → он открывается у обоих.
7. Жмёте play/пауза/перематываете — у второго повторяется автоматически. Пишете в чат.

Готово. Приятного просмотра! 🎬

---

## Если что-то не так

- **Видео не играет / чёрный экран** — формат не mp4 (H.264). Сконвертируй в mp4.
- **«У этого фильма не указана ссылка»** — забыл вставить `videoUrl` в `movies.json`.
- **Сайт долго грузится при первом заходе** — Render «проснулся» после сна, подожди минуту.
- **Перемотка не синхронится** — обнови обе вкладки/страницы (переподключение к комнате).
- **Друг не видит твою комнату** — проверьте, что оба ввели один и тот же код и сайт
  открыт по адресу Render (а не localhost).
- **10 ГБ в R2 кончились** — удали старые фильмы из корзины или подключи платный тариф.
